# DevLoop AI Project ZIP and Logo Design

## Goal

Extend DevLoop AI so a final iteration can include a downloadable, runnable multi-file project instead of only one code block. Add a polished project logo that fits the existing dark cyan-and-violet interface. Keep the implementation small enough to verify and deploy before the July 19, 2026 submission deadline.

## User Experience

When a requirement asks DevLoop to create or set up a project, Gemini returns a project-file manifest alongside the primary preview code. The final result shows the number of generated files and a **Download project ZIP** button. The archive uses a safe, readable filename such as `devloop-angular-project-v3.zip` and preserves the manifest's folder structure.

Single-file requests continue to work exactly as they do now. Their result keeps the code preview and copy action and does not show a misleading project download.

The header replaces the CSS-only diamond with a generated DevLoop symbol. The wordmark remains live HTML text for sharp rendering and accessibility. The same symbol becomes the favicon and appears in the repository README.

## Approaches Considered

### Structured manifest generated by Gemini and zipped in the browser

This is the selected approach. Gemini returns explicit file paths and contents. The Angular client validates the received manifest and uses JSZip to build the archive locally. It avoids a second download endpoint, keeps the API stateless, and makes the archive available immediately after a completed run.

### Backend-generated archive

This centralizes archive creation but adds a binary endpoint, temporary state or repeated request payloads, and more deployment complexity. It is unnecessary for the hackathon MVP.

### Parse Markdown code fences

This requires guessing filenames and folder boundaries from prose. It is too fragile for a button labelled as a runnable project download.

## Data Model

`GenerationResult` gains an optional `files` property:

```ts
interface ProjectFile {
  path: string;
  content: string;
}

interface GenerationResult {
  code: string;
  language: string;
  changes: string[];
  files?: ProjectFile[];
}
```

The primary `code` field remains required because it powers the existing iteration comparison and code preview. For multi-file output it contains the most representative entry file or a concise project overview. Existing persisted runs without `files` remain compatible.

## Gemini Contract

The generation and improvement prompts request the existing fields plus an optional `files` array. They instruct Gemini to include a manifest for project scaffolding, repository setup, or explicitly multi-file requirements. A runnable Angular scaffold should include the package manifest, Angular configuration, TypeScript configuration, application entry point, component files, styles, tests, and a README with install and run commands.

The Zod schema accepts no more than 50 files, no file larger than 100,000 characters, and no manifest larger than 500,000 characters in total. Each path must be relative and normalized. Absolute paths, empty path segments, backslashes, null bytes, `.` segments, and `..` segments are rejected.

The demo provider remains deterministic and single-file. This keeps the reliable judging walkthrough unchanged; multi-file ZIP generation is a Gemini-live feature.

## ZIP Creation

A small Angular `ProjectDownloadService` receives validated files, adds them to JSZip, generates a Blob, and triggers an `<a download>` action. The service derives a conservative archive name from the requirement and selected iteration, falling back to `devloop-project-vN.zip`.

The UI displays the download action only when the selected iteration contains at least one project file. It shows the file count next to the button. The download runs entirely in the browser; generated files are not executed, uploaded, or stored by DevLoop.

## Logo Direction

Generate one square symbol-only logo with an abstract continuous loop made from two interlocking paths and subtle circuit-node details. Use luminous cyan and violet on deep navy, with crisp geometry, restrained glow, no words, no letters, no mockup, and no watermark. The mark should remain recognizable at favicon size.

The selected raster asset is copied into `web/public/branding/`. A full-size PNG is used in the README and a smaller optimized PNG is used in the application header and favicon. The live `DevLoop AI` wordmark stays beside it.

## Error Handling and Safety

- Invalid manifests fail structured validation and receive the provider's existing one-retry behavior.
- ZIP creation errors produce an accessible inline message and leave the generated code visible.
- File paths are validated before they reach JSZip to prevent path traversal.
- The browser never receives or stores the Gemini API key.
- Generated projects are labelled as AI-generated and are not claimed to have been executed or production-verified.
- Archive generation is disabled for empty or oversized manifests.

## Testing

Backend tests cover valid multi-file generation output, unsafe-path rejection, file-count and size limits, and Gemini prompt instructions. Existing deterministic loop tests remain unchanged.

Frontend tests cover ZIP contents and nested paths, filename sanitization, conditional download-button rendering, current single-file compatibility, and an accessible download error. Existing workflow, persistence, and score tests must continue to pass.

Verification includes all API tests, all Angular tests, both production builds, a browser check of the logo and download action, inspection of the downloaded archive, and a live Gemini run once AI Studio billing reports usable credits.

## Deployment

The change ships through the existing GitHub `master` branch and Render service. The new `jszip` dependency is included in the npm lockfile. The public service remains on Render's free instance. If Gemini billing is not usable by the recording deadline, the app can be switched back to Demo mode; the ZIP button will then remain hidden rather than presenting fake multi-file output.

## Scope Boundaries

This version does not execute generated projects, run `npm install`, validate framework compilation, offer individual file editing, persist archives, or generate GitHub repositories. It produces a structured archive for users to inspect and run locally.
